Study Diary for Code II

Classes

  • parameters + methods
  • class vs object: first create class x, then you can create objects of class x (instances)
  • conventions:
    • each class gets their own file -> subclasses can stay in the superclass file
    • file name = class name
    • method names don't repeat class name (scale instead of scaleVector)
    • class name is singular

Subclasses

  • allows for specifications
  • "class x extends y" -> class x inherits all parameters and methods of class y

Namespaces

  • variables, functions and classes can be shared throughout a namespace
  • add export before a variable, function or class to make it public to the namespace

For in/For of Loop

  • for in loop loops through the keys/indexes of an array
  • for of loop loops through the elements of an array

Time signals

  • window.setTimeout(handler,time,args...)
  • window.setInterval(handler,time,args...)
  • window.requestAnimationFrame(handler)

Four principles of object orientation

  • Abstraction: breaks a concept down to the most basic functions and features
  • Encapsulation: is as independent as possible
  • Inheritance: traits can be inherited (super + subclasses)
  • Polymorphism: a class can have many shapes (super + subclasses)

Five questions to ask for each object type

  • What does it have?
  • What can it do?
  • What does it know?
  • Who holds it?
  • What is it?

Two commandments for each part of a system

  • Each part can only be as intelligent as necessary
  • Each part has to be as dumb as possible

"Abstract"

  • Abstract classes: only a concept/an idea, no instances can be created
  • Abstract methods: basically a placeholder, each subclass has to define the method

Static methods

  • can be called on the object class, not on objects of the class

Visibility

  • public: everything can access this
  • private: only objects of the same class can access this
  • protected: objects of the same class and its subclasses have access
  • readonly: can be combined with the other modificators, once set the value is non changeable (like const for other variables)

Method types: Set & Get

  • Set: gets called when changing a value of the object, for example allowing it to interfere
  • Get: used to get a value that for example is only calculated when needed
